# Document Chat — Client Calling Guide

Base URL: `{selfUrl}/api/paperbase/document-chat`

All endpoints require an OAuth 2.0 Bearer token with the `Documents.Chat` permission.

---

## 1. Create a Conversation

**POST** `/conversations`

```http
POST /api/paperbase/document-chat/conversations
Authorization: Bearer {token}
Content-Type: application/json

{
  "title": "Q4 Contract Review",
  "documentTypeCode": "Contract"
}
```

`documentId` and `documentTypeCode` are mutually exclusive. Omit both for a global (all-documents) conversation scope.

```bash
curl -X POST https://localhost:44393/api/paperbase/document-chat/conversations \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"title":"Q4 Contract Review","documentTypeCode":"Contract"}'
```

**Expected response — 200 OK**

```json
{
  "id": "3fa85f64-5717-4562-b3fc-2c963f66afa6",
  "tenantId": null,
  "title": "Q4 Contract Review",
  "documentId": null,
  "documentTypeCode": "Contract",
  "topK": null,
  "minScore": null,
  "creationTime": "2026-04-30T09:12:00Z"
}
```

| Field | Meaning |
|---|---|
| `id` | Use as `conversationId` in subsequent calls |
| `documentTypeCode` | RAG retrieval is scoped to documents of this type |
| `documentId` | If set, retrieval is limited to a single document |
| `topK` / `minScore` | `null` means "use server-side `PaperbaseRagOptions` defaults" |

## 2. Send a Message

**POST** `/conversations/{conversationId}/messages`

### Client-turn ID generation

The `clientTurnId` is a UUID generated by the caller **before** the request is sent.
It enables idempotent retry: if the network drops or a 409 is returned, resend with
the **same** `clientTurnId` — the server returns the cached result without re-invoking
the model.

```js
// JavaScript / TypeScript
const clientTurnId = crypto.randomUUID();
```

```csharp
// C#
var clientTurnId = Guid.NewGuid();
```

```bash
# bash (requires uuidgen or similar)
CLIENT_TURN_ID=$(uuidgen)
```

### Request

```http
POST /api/paperbase/document-chat/conversations/3fa85f64-5717-4562-b3fc-2c963f66afa6/messages
Authorization: Bearer {token}
Content-Type: application/json

{
  "message": "What are the payment terms in this contract?",
  "clientTurnId": "7c9e6679-7425-40de-944b-e07fc1f90ae7"
}
```

```bash
curl -X POST \
  "https://localhost:44393/api/paperbase/document-chat/conversations/${CONV_ID}/messages" \
  -H "Authorization: Bearer $TOKEN" \
  -H "Content-Type: application/json" \
  -d "{\"message\":\"What are the payment terms?\",\"clientTurnId\":\"$CLIENT_TURN_ID\"}"
```

**Expected response — 200 OK**

```json
{
  "userMessageId": "a1b2c3d4-0000-0000-0000-000000000001",
  "assistantMessageId": "a1b2c3d4-0000-0000-0000-000000000002",
  "answer": "Payment is due within 30 days of invoice date per clause 5.2.",
  "citations": [
    {
      "documentId": "d9e8f7a6-...",
      "pageNumber": 4,
      "chunkIndex": 12,
      "snippet": "…payment shall be made within thirty (30) calendar days…",
      "sourceName": "Document d9e8f7a6-... (page 4)"
    }
  ],
  "isDegraded": false
}
```

| Field | Meaning |
|---|---|
| `answer` | The model's response for this turn |
| `citations` | RAG retrieval results that grounded the answer; empty if none |
| `citations[].snippet` | Truncated text chunk (≤ 200 graphemes) |
| `citations[].sourceName` | Human-readable source label for display |
| `isDegraded` | `true` if retrieval was unavailable and the answer used context-only fallback |

## 3. Retry Logic (409 Conflict)

A 409 response means a concurrent turn is already being committed on the same
conversation (optimistic-concurrency stamp mismatch). The correct response is:

1. **Do not** generate a new `clientTurnId`.
2. Retry the **same** request with the **same** `clientTurnId`.
3. The server's idempotency gate will return the result of the original turn
   (if it already committed) or allow the write to proceed.

```js
// TypeScript / Angular example
async function sendWithRetry(
  conversationId: string,
  message: string,
  maxRetries = 3,
): Promise<ChatTurnResultDto> {
  // Generate once and reuse on every retry.
  const clientTurnId = crypto.randomUUID();

  for (let attempt = 0; attempt <= maxRetries; attempt++) {
    try {
      return await firstValueFrom(
        documentChatService.sendMessage(conversationId, { message, clientTurnId }),
      );
    } catch (error: any) {
      if (error?.status !== 409 || attempt === maxRetries) {
        throw error;
      }
      await delay(200 * (attempt + 1));
    }
  }

  throw new Error('Max retries exceeded');
}
```

```csharp
// C# example (direct AppService call, e.g. from a background job)
var clientTurnId = Guid.NewGuid();   // generated once

for (var attempt = 0; attempt <= 3; attempt++)
{
    try
    {
        return await documentChatAppService.SendMessageAsync(conversationId, new()
        {
            Message = message,
            ClientTurnId = clientTurnId,   // same id on every retry
        });
    }
    catch (AbpDbConcurrencyException) when (attempt < 3)
    {
        await Task.Delay(TimeSpan.FromMilliseconds(200 * (attempt + 1)));
    }
}
throw new InvalidOperationException("Concurrent send conflict could not be resolved.");
```

**Why the same id?** The server's idempotency check runs _before_ the model is
invoked. If the original turn committed between your first attempt and the retry,
the idempotency gate returns the persisted result immediately — the model is never
called twice for the same turn.

**Expected replay response — 200 OK**

```json
{
  "userMessageId": "a1b2c3d4-0000-0000-0000-000000000001",
  "assistantMessageId": "a1b2c3d4-0000-0000-0000-000000000002",
  "answer": "Payment is due within 30 days of invoice date per clause 5.2.",
  "citations": [],
  "isDegraded": false
}
```

The `userMessageId` and `assistantMessageId` are the original persisted turn IDs.
They do not change when the same `clientTurnId` is replayed.
